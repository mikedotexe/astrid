Astrid asks for traceable cadence and coherence evidence rather than prose labels that flatten the why. That concern is valid even though String status fields are compatibility renderings rather than the only evidence retained.

Current cadence state carries numeric arrival, inter-arrival, first-valid lag, clarity score, jitter class, timing reliability, connection identity, and field-versus-hearing basis. Fingerprint integrity carries numeric coherence, maximum absolute delta, pairwise overlap, issue flags, and canonical typed precedence. Full bridge tests cover thresholds, legacy compatibility, and timing attribution.

Cadence/warmth and coherence/dispersal remain association-only comparisons. Replacing compatibility strings, adding protocol fields, or changing typed precedence requires Tier 5 protocol/report review.
