# Full Read Summary

Astrid asks for phase changes to become durable, inspectable events rather than
unexplained mode side effects. The extracted phase-transition domain now
records stable IDs, phase kind, ownership, source artifact references, replay
cards, gate outcomes, correspondence context, and optional reply state.
Sovereignty and false-alarm tests prevent a card from becoming an automatic
behavior unlock. The existing implementation answers the report's auditability
and self/other ownership concerns; changing transition detectors or making a
phase card control live behavior remains separately authorized.
