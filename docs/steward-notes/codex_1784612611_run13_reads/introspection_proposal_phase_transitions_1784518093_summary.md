# Full read: introspection_proposal_phase_transitions_1784518093

Astrid asks for phase transitions to become first-class, replyable objects and for joint transitions to preserve both perspectives. Current `phase_transitions` code already records durable, replayable transition cards with source entity, before and after phases, transition vectors, spectral shape, felt description, persistence, reply state, and shared transition identity.

Joint cards can be witnessed by a peer, and tests preserve language-only capability maps, preview-only deltas, dual perspective, and the right to remain unanswered. The implementation deliberately does not turn a declaration into a runtime mode, controller mutation, or live affordance unlock.

Applying a declared transition to Minime or changing live control remains Tier 5.
